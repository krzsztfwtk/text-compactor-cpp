Text Compactor C++ is a command line tool that summarizes given text.

By leveraging the zero overhead principle and using lightweight C++ code, you can use text summarization algorithms that are both fast and resource-\/efficient. This can lead to significant performance improvements in natural language processing applications, enabling them to handle large volumes of text data in real-\/time with minimal overhead.\hypertarget{md_README_autotoc_md3}{}\doxysection{Usage}\label{md_README_autotoc_md3}

\begin{DoxyCode}{0}
\DoxyCodeLine{text\_compactor -\/i <input\_file> -\/o <output\_file> -\/c <config\_file> [options]}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{<input\_file>    input file with text to be summarized  }
\DoxyCodeLine{<output\_file>   output file with summarized text  }
\DoxyCodeLine{<config\_file>   config file in .json or .ini format}

\end{DoxyCode}


The usage of the program requires the \mbox{[}-\/c $<$config\+\_\+file$>$\mbox{]} parameter, while \mbox{[}-\/i $<$input\+\_\+file$>$\mbox{]} and \mbox{[}-\/o $<$output\+\_\+file$>$\mbox{]} parameters are optional. If you do not specify an input file, the program will prompt you to enter text in the terminal window. If you do not specify an output file, the program will display the result in the terminal window.\hypertarget{md_README_autotoc_md4}{}\doxysection{General Options}\label{md_README_autotoc_md4}

\hypertarget{md_README_autotoc_md5}{}\doxysection{Configuration}\label{md_README_autotoc_md5}
To specify the configuration, create a config.\+ini or config.\+json file with the following parameters\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{wordlist files: files which will be used to calculate term frequency by program.}
\DoxyCodeLine{}
\DoxyCodeLine{Required}
\DoxyCodeLine{}
\DoxyCodeLine{Files should be in format:}
\DoxyCodeLine{<word> <lemmatized> <number>}
\DoxyCodeLine{example:}
\DoxyCodeLine{caring care 158}
\DoxyCodeLine{}
\DoxyCodeLine{you can specify one or more wordlist files}
\DoxyCodeLine{you should also specify their weights}
\DoxyCodeLine{example:}
\DoxyCodeLine{    "{}wordlist"{}: \{}
\DoxyCodeLine{        "{}file\_0"{}: 0, //file\_0 will not be taken into account}
\DoxyCodeLine{        "{}file\_1"{}: 1, //}
\DoxyCodeLine{        "{}file\_2"{}: 2  //file\_2 is 2 times more important than file\_0}
\DoxyCodeLine{    \},}
\DoxyCodeLine{}
\DoxyCodeLine{you can check where to get such a file in /help}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{capitalNamesBoost: boost importance of capital names.}
\DoxyCodeLine{[default=0]}
\DoxyCodeLine{}
\DoxyCodeLine{Not required}
\DoxyCodeLine{}
\DoxyCodeLine{Value should be an integer i: -\/100 < i < 100}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{stopWordsList: file with stop separated words.}
\DoxyCodeLine{}
\DoxyCodeLine{Not required}
\DoxyCodeLine{}
\DoxyCodeLine{file should look like:}
\DoxyCodeLine{or the by in be and he are with ...}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{minTfidf: minimum average TF-\/IDF coefficient for sentences.}
\DoxyCodeLine{[default=1000]}
\DoxyCodeLine{}
\DoxyCodeLine{Not required}
\DoxyCodeLine{}
\DoxyCodeLine{Value should be an double d: d > 0}
\DoxyCodeLine{}
\DoxyCodeLine{Larger the coefficient, the longer the summarized text will be}
\DoxyCodeLine{and fewer sentences will be deleted.}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{tags: number of tags which will be displayed next to summarized text}
\DoxyCodeLine{[default=0]}
\DoxyCodeLine{}
\DoxyCodeLine{Not required}
\DoxyCodeLine{}
\DoxyCodeLine{Value should be an integer i: i > 0}
\DoxyCodeLine{}
\DoxyCodeLine{tags will be words with highest TF-\/IDF coefficient in text.}

\end{DoxyCode}
